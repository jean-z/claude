# 合并当前分支到 UAT5

将当前分支合并到 uat5 分支的自动化流程。

## 使用方法

```
/merge-to-uat5 [分支名]
```

**参数说明：**
- `分支名`（可选）：指定要合并的分支，默认为当前分支

## 执行步骤

请按照以下步骤执行合并操作：

### 步骤 1：获取当前分支信息

```bash
# 获取当前分支名
git branch --show-current

# 检查当前是否有未提交的更改
git status --porcelain
```

**注意：** 如果有未提交的更改，需要先提示用户处理（提交或暂存）。

### 步骤 2：保存当前分支名并切换到 uat5

```bash
# 切换到 uat5 分支
git checkout uat5
```

### 步骤 3：拉取 uat5 最新代码

```bash
# 拉取远程最新代码
git pull origin uat5
```

### 步骤 4：合并源分支

```bash
# 合并源分支到 uat5
git merge <源分支名> --no-edit
```

### 步骤 5：检查合并结果

**情况 A：合并成功（无冲突）**

输出合并成功的提示，并询问用户是否推送：

```
✅ 合并成功！无冲突。

是否推送到远程 uat5 分支？
- 输入 `yes` 或 `y` 推送
- 输入 `no` 或 `n` 不推送（稍后手动推送）
```

如果用户确认推送：
```bash
git push origin uat5
```

**情况 B：合并有冲突**

1. 列出冲突文件：
```bash
git diff --name-only --diff-filter=U
```

2. 显示冲突详情，并提示用户：
```
⚠️ 发现冲突，请解决以下文件的冲突：

- 文件1路径
- 文件2路径
- ...

冲突解决完成后，请输入：
- `resolved` 或 `done` - 继续完成合并
- `abort` - 取消本次合并
```

3. 等待用户解决冲突后，执行：
```bash
# 检查是否还有未解决的冲突
git diff --check

# 如果没有冲突标记，添加所有文件并完成合并
git add .
git commit --no-edit
```

4. 询问是否推送：
```
✅ 冲突已解决，合并完成！

是否推送到远程 uat5 分支？
```

**情况 C：合并被取消**

如果用户选择 `abort`：
```bash
git merge --abort
```

### 步骤 6：切回原分支

无论合并成功与否，最后都要切回原分支：

```bash
git checkout <原分支名>
```

## 注意事项

1. **工作区必须干净** - 合并前确保没有未提交的更改
2. **冲突处理** - 遇到冲突时，等待用户手动解决后再继续
3. **安全回退** - 任何时候出错都可以通过 `git merge --abort` 取消
4. **最后切回** - 无论成功或失败，最后都要切回原分支
5. **推送确认** - 推送到远程前需要用户确认

## 错误处理

| 错误情况 | 处理方式 |
|----------|----------|
| 工作区有未提交更改 | 提示用户先提交或暂存 |
| uat5 分支不存在 | 提示用户先创建分支 |
| 拉取失败 | 检查网络连接，重试或提示用户 |
| 合并冲突 | 列出冲突文件，等待用户解决 |
| 推送失败 | 检查权限，提示用户手动处理 |
