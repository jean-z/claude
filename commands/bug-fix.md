# 系统化 Bug 修复

请使用 bug-fix Skill 执行系统化 Bug 修复流程。

用户提供的 Bug 信息：$ARGUMENTS

## 执行流程

严格按照以下 6 个阶段执行，每一步必须完成后才能进入下一步：

1. **Phase 1: 接收报错** - 收集报错信息，理解 Bug 外在表现
2. **Phase 2: 定位根因** - 追踪调用链，找到根本原因（不是表面现象）
3. **Phase 3: 制定方案** - 设计最小改动方案，不做无关重构
4. **Phase 4: 执行修复** - 按方案修改代码
5. **Phase 5: 验证修复** - 运行测试或构建确认修复有效
6. **Phase 6: 总结经验** - 输出结构化总结，记录根因类型和预防措施

## 注意事项

- 如果用户提供的报错信息不足，主动追问：触发条件、期望行为 vs 实际行为、控制台报错
- **禁止**不加分析就用 try-catch 掩盖问题
- **禁止**顺手重构无关代码
- **禁止**跳过验证步骤
- **必须**全局扫描同类问题
