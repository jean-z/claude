# GitLab MR 代码审查

请对 MR 编号 $ARGUMENTS 进行全面的代码审查。

## 执行步骤

1. **获取 MR 基本信息**
   ```bash
   glab mr view $ARGUMENTS
   ```

2. **获取代码改动**
   ```bash
   glab mr diff $ARGUMENTS
   ```

3. **获取现有评论**
   ```bash
   glab api "projects/h5%2Fbiz-shopping-mobile/merge_requests/$ARGUMENTS/notes"
   ```

## 审查维度

请从以下维度进行审查并输出结构化报告：

### 1. 基本信息
- 标题、作者、状态、目标分支

### 2. 代码质量
- 逻辑正确性
- 代码可读性
- 命名规范
- 重复代码

### 3. 安全性
- 潜在安全漏洞
- 敏感信息泄露
- 输入验证

### 4. 性能
- 性能影响
- 内存泄漏风险
- 不必要的计算

### 5. 最佳实践
- 是否符合项目规范
- 错误处理是否完善
- 边界条件处理

### 6. 建议改进
- 具体的改进建议
- 可选的优化方案

## 输出格式

```markdown
## Code Review: MR {编号}

### 📋 基本信息
| 项目 | 内容 |
|------|------|

### 📝 改动概述
简要描述改动内容

### ✅ 优点
列出做得好的地方

### ⚠️ 问题与建议
列出问题和改进建议

### 📊 评分总结
| 维度 | 评分 | 说明 |
|------|------|------|

### 结论
给出是否建议合并的结论
```

**重要**: 使用中文输出所有内容。
